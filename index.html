Project Overview

This project combined the major sensing, actuation, and control systems developed in ME405 into one autonomous Romi robot capable of completing an obstacle course. Our final design brought together line following, bump detection, encoder-based motion control, and IMU-based state estimation into a single multitask robot system.

The goal of the project was to take the individual labs completed throughout the quarter and integrate them into one working mechatronic platform. By the end of the project, our robot was able to follow a printed path, respond to obstacles, perform turning maneuvers, and use multiple sensors together to improve navigation and consistency.

Hardware / Mechanical & Electrical Design

Base Platform

Our robot was built on the standard Romi platform used throughout the ME405 lab sequence. The base system included the Romi chassis, DC motors, wheel encoders, a Nucleo L476RG microcontroller, a modified Shoe of Brian, and a BNO055 IMU breakout board.

Added Sensors and Components

To improve obstacle-course performance, we added a QTRX-MD-07A 7-channel analog reflectance sensor array for line detection and right and left bumper switch assemblies for wall contact sensing. These additions allowed the robot to follow the course line while also detecting collisions or boundary contact during navigation.

Mechanical Layout

The reflectance sensor array was mounted near the front underside of the robot using stand-offs so that it sat just above the ground for reliable line sensing. The bump sensors were mounted at the front of the robot in a position that allowed them to contact walls before the line sensor, helping protect the sensor array from impact. This layout was chosen to improve sensor reliability while keeping the robot compact and functional.

Electrical Integration

All components were wired into the Nucleo using a common ground system. Power was distributed through VIN and GND, while motors, encoders, the line sensor array, IMU, and bump sensors were connected to their assigned pins on the microcontroller. This wiring arrangement allowed us to combine motor control, encoder feedback, line sensing, inertial measurement, and bumper input into one integrated robot system.

Official Time

2:00

Our robot successfully completed the main required tasks of the final track, including line following, turning, wall interaction, and overall path navigation. The only portion of the course we did not complete was the bonus cup-pushing task. Our official completion time for the track was exactly 2 minutes.

In our motor tuning and control development, the best step-response behavior came from using a proportional gain of Kp = 2.9 and an integral gain of Ki = 0.1 along with motor gain of K = 0.2 and set point of S = 120 counts/s.

One major challenge in tuning the robot was that changing the control values often caused unpredictable motor behavior. Small adjustments to the gains or the setpoint could make the response less stable, less repeatable, or more difficult to control during the full track run. Because of this, once we found a set of values that gave acceptable performance, we tried to keep them consistent rather than continue making aggressive tuning changes right before the demonstration.

Overall, the robot demonstrated the successful integration of the major subsystems developed throughout the quarter. Even though the cup task was not completed, the robot was still able to perform the core navigation tasks required by the project and complete the track within the official time limit.

Code / Software Architecture

Our software was organized as a multitask robot control system. At a high level, the code handled user interaction, motor control, encoder feedback, line sensing, bump sensing, navigation, and state estimation. The project was built from the earlier labs and then combined into one final program for the obstacle course.
